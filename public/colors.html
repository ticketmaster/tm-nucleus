Shades

Shades are used as visual cues for interactivity, such as for hovering, when an element is disabled, etc.

Shades should not be used statically — that is, they should only ever be seen when the user has initiated an action. You should not be using any shades when designing static elements of your component.

Shade Usage
base the default color
dark for hover states
light for indicating that something is outside of your selection criteria
muted for disabled states
background for use as backgrounds

Note that for most of these shades, their relationships and usage are well-defined and should not have any deviations. For example, the dark shade should always be used for a visual cue of hovering over an element.

Palette Customization

Nucleus has built-in support for Ticketmaster and Live Nation palettes. These are located in the /themes directory, and are named tm.scss and ln.scss, respectively.

Switching between TM and LN is just a matter of changing which theme to import.

/* TM theme */
@import '[path/to/nucleus]/tm';
@import '[path/to/nucleus]/base/all';
@import '[path/to/nucleus]/mixins/all';
@import '[path/to/nucleus]/components/all';
@import '[path/to/nucleus]/states/all';

/* LN theme */
@import '[path/to/nucleus]/ln';
@import '[path/to/nucleus]/base/all';
@import '[path/to/nucleus]/mixins/all';
@import '[path/to/nucleus]/components/all';
@import '[path/to/nucleus]/states/all';

To apply custom colors, refer to ln.scss as your model. You will need to merge any custom colors into Nucleus's palette, then apply them via the nc-color() function.

Important Note

All of your color definitions should be abstracted in your theme file. This is to support branding at the app level. (Themes are not always 1:1; the fact that everything that's blue in the TM theme does not necessarily mean that it is red in the LN theme.)

To use your custom theme, import it before anything else from Nucleus.

/* Custom theme */
@import '[path/to/custom]/your-theme';
@import '[path/to/nucleus]/base/all';
@import '[path/to/nucleus]/mixins/all';
@import '[path/to/nucleus]/components/all';
@import '[path/to/nucleus]/states/all';

Code Reference

nc-color() Function themes/_shared.scss

Returns hex value from palette.

Parameters Default Description Required
$name   Name of color. (e.g., sierra.)
$shade base Name of shade. (e.g., dark.)  
$map $nc-palette SASS map that contains palette.   
$var1: nc-color(sierra);           /* #475058 */
$var2: nc-color(pacific, muted);   /* #bfe6f7 */

nc-get-darker() Function themes/_shared.scss

Given a color, returns a new color mixed with a percentage of white (#000).

Parameters Default Description Required
$color   Hex value.
$percent 25% Percentage of white to mix in with the given $color.   
$darker: nc-get-darker(#19ab20);  /* #138018 */

nc-get-lighter() Function themes/_shared.scss

Given a color, returns a new color mixed with a percentage of white (#fff).

Parameters Default Description Required
$color   Hex value.
$percent 25% Percentage of white to mix in with the given $color.   
$lighter: nc-get-lighter(#f00, 50%);  /* #ff8080 */
