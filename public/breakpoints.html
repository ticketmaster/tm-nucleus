Declaring Breakpoints

In SASS, declare your breakpoints within the elements affected. Do not declare media queries as another code block separate from the original context.

/* BAD */
.class1 {
  font-size: 12px;
}
.class2 {
  padding: 50px;
}

/* a bunch of other code between these lines */

@include nc-breakpoint($bp-md) {
  .class1 {
    font-size: 18px;
  }
  .class2 {
    padding: 100px;
  }
}

Declaring your breakpoints in this way places the modifications too far from the original definitions. The following is much easier to parse.

/* GOOD */
.class1 {
  font-size: 12px;

  @include nc-breakpoint($bp-md) {
    font-size: 18px;
  }
}

.class2 {
  padding: 50px;

  @include nc-breakpoint($bp-md) {
    padding: 100px;
  }
}

Code Reference

nc-breakpoint() Mixin mixins/_breakpoint.scss

Generates media queries.

Parameters Default Description Required
$min false Specify minimum device width which styles should apply  
$max false Specify maximum device width which styles should apply  

By default, if only one parameter is passed, it is assigned to the $min value.

The mixin takes px values and converts them to em units, which is the preferred way to write media queries.

Notice in the max-width example below that we actually pass in ($max: $bp-lg - 1) because we want the styling rules to apply to anything below the max value. (We don't want to actually include the max value as part of the declaration.) 

/* defining a min-width breakpoint */
.some-block {
  margin-top: 15px;

  @include nc-breakpoint($bp-md) {
    margin-top: 40px;
  }
}

/* defining a max-width breakpoint */
.another-block {
  [...]

  @include nc-breakpoint($max: $bp-lg - 1) {
    font-size: 20px;
  }
}
